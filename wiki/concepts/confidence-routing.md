# Confidence-based Email Routing

AI 기반 자동 이메일 응답 시스템에서 모델의 확신도(Confidence Score)에 따라 처리 경로를 결정하는 패턴. [[keychron]] CS 자동화에 적용된다.

## 라우팅 임계값

| 확신도 구간 | 경로 | 처리 방식 |
|-------------|------|-----------|
| **>= 0.85** | 자동 발송 | 사람 개입 없이 즉시 응답 |
| **0.70 ~ 0.84** | 승인 큐 | 담당자가 확인 후 발송/수정 |
| **< 0.70** | 에스컬레이션 | 전문 담당자에게 전달 |

### 임계값 설정 근거

- **0.85**: 오답률 5% 미만 목표 (실측 기반 튜닝)
- **0.70**: 답변 품질이 불안정해지는 경계선
- 임계값은 카테고리별로 조정 가능 (AS 문의는 더 높게 설정)

## 이중 엔진 아키텍처

### Claude CLI (Primary Engine)

- 복잡한 문맥 이해, 정책 준수, 다국어 지원
- API 호출 비용이 발생하므로 고확신 케이스에 우선 적용
- 응답 품질이 높고 환각(hallucination)이 적음

### Ollama (Secondary Engine)

- 로컬 실행으로 비용 절감
- 간단한 FAQ, 배송 조회 등 정형화된 응답에 적합
- Claude CLI가 불가할 때 폴백(fallback) 역할

### 엔진 선택 로직

1. 수신 이메일 분류 (카테고리, 복잡도)
2. 정형 FAQ 매칭 시 → Ollama 우선
3. 비정형 또는 고난이도 → Claude CLI
4. 두 엔진의 확신도를 비교하여 높은 쪽 채택

## Scope-based Entity Isolation

자동 응답 시스템에서 데이터 접근 범위를 엄격히 분리하는 원칙.

### Shared Scope (공유 데이터)

모든 자동 응답에서 접근 가능:

- 상품 정보 ([[product-code-hierarchy|G-code]] 기반)
- 배송 정책, 반품/교환 규정
- 일반 FAQ
- 공개 프로모션 정보

### Personal Scope (개인 데이터)

해당 고객의 요청에만 접근 가능:

- 주문 내역 (주문번호 매칭 필수)
- AS 이력
- 개인 배송 정보
- 결제 정보 (조회만, 노출 불가)

### 격리 규칙

- Personal Scope 데이터는 요청자 인증 후에만 접근
- 다른 고객의 데이터를 참조하거나 노출하는 것은 절대 금지
- 로그에 개인정보가 남지 않도록 마스킹 처리

## 품질 모니터링

### 자동 발송 사후 점검

- 자동 발송된 응답의 5%를 랜덤 샘플링하여 품질 검토
- 오답률이 임계값을 초과하면 확신도 기준 상향 조정
- 고객 만족도 피드백과 확신도 간 상관관계 추적

### 승인 큐 효율성

- 승인 큐의 평균 처리 시간 모니터링
- 승인율이 90% 이상이면 임계값 하향 검토
- 반복 수정 패턴을 학습하여 프롬프트 개선
