# GFA OPS AI (광고 자동 최적화)

## 개요

GFA OPS AI는 네이버 GFA(성과형 디스플레이 광고)의 예산 배분과 캠페인 ON/OFF를 자동으로 제어하는 AI 엔진이다. [[tbnws-ad]] 대시보드에 통합되어 운영되며, 네이버 GFA 내부 API를 직접 호출하여 실시간 최적화를 수행한다.

## 기술 스택

| 구분 | 기술 |
|------|------|
| API 클라이언트 | Python requests + session |
| 인증 | Cookie + XSRF Token |
| 데이터 | MySQL (TBNWS_AD) |
| 실행 환경 | EC2 SSE 모드 |

## 핵심 컴포넌트

### OpsGfaApi

네이버 GFA 내부 API와 통신하는 클라이언트 클래스:
- `requests.Session`을 사용한 세션 유지
- Cookie 기반 인증 + XSRF 토큰 자동 처리
- 캠페인 예산 변경, ON/OFF 제어, 소재 관리 API 호출

### 6차원 효율 스코어

각 GFA 캠페인을 6가지 차원으로 평가한다 (0~100점):

| 차원 | 설명 |
|------|------|
| adj_ROAS | 보정 ROAS (광고비 대비 전환매출) |
| 볼륨 | 광고 노출/클릭 규모 |
| 추세 | 최근 성과 상승/하락 방향 |
| 페이싱 | 예산 소진 속도 적정성 |
| 포화도 | 타겟 오디언스 포화 정도 |
| 퍼널밸런스 | TOF/MOF/BOF 비중 균형 |

### 피로도 감지

소재(크리에이티브)의 성과 열화를 자동 감지한다:
- **단기 비교** — 최근 7일 vs 직전 7일의 CTR/CPM 변화
- **장기 베이스라인** — 30일 평균 대비 이탈 감지
- 피로도가 높은 소재는 교체 또는 OFF 권고

### 낭비 소재 제거

전환 0건인 소재를 자동 식별:
- 광고비만 소진하고 전환이 없는 소재 목록 추출
- 월간 절감 가능액 산출
- 자동 OFF 또는 운영자 확인 후 처리

## 자동화 워크플로

### 예산 재배분 제안
1. 6차원 효율 스코어 산출
2. 캠페인별 증액/감액/유지 판단
3. 시뮬레이션 결과와 함께 제안 생성
4. 운영자 승인 후 자동 반영 (또는 위임 모드)

### 실시간 알림 (6종)
1. **예산 과다 소진** — 일 예산 소진율 이상
2. **ROAS 하락** — 기준 대비 급격한 하락
3. **CPM 급등** — 입찰 경쟁 과열 감지
4. **CartRate 하락** — 장바구니 전환율 이상
5. **지출 예측 초과** — 월말 예산 초과 예측
6. **GFA-SS 괴리** — GFA 보고 매출과 스마트스토어 실매출 차이

## 운영 모드

- **EC2 SSE 모드** — Server-Sent Events 기반 실시간 스트리밍
- 효율 스코어, 예산 제안, 피로도 분석 등은 SSE 모드에서만 동작
