# Claude 비서 시스템 — 매니저 가이드

## 개요

ToBe Networks 매니저의 업무 비서 시스템. Claude Desktop Cowork 모드에서 노션 업무일지를 자동 관리한다.

- **하루 3회 자동 체크인**: 오전(9시), 오후(14시), 퇴근(17:30)
- **실시간 업무 추적**: 매니저가 상태를 말하면 즉시 노션 업데이트
- **주간 리포트**: 금요일 자동 생성
- **프로덕트 카드 연동**: [브랜드명] 접두사로 프로젝트 DB 자동 카드 생성

## 사용법

Claude Desktop 앱의 **Cowork** 모드에서 노션 가이드 링크를 주면 시작:
1. 이름 확인 → 기존 업무일지 파악
2. 스케줄러 3개 등록 (1회만)
3. 이후 매일 자동 체크인

## DB 구조

| DB | 용도 | data source |
|----|------|-------------|
| daily work log | 읽기 전용 (과거 업무 파악) | `collection://208c0e21-8a4f-8113-b68c-000b25ac520f` |
| ToBe Daily Report | 쓰기 전용 (정제된 리포트) | `collection://fe6bf7ad-eb38-4e33-b958-429ad423e0a7` |

**절대 규칙**: 기존 `daily work log`에는 쓰지 않음. 새 리포트는 `ToBe Daily Report`에만 생성/수정.

## 하루 3회 체크인

### 오전 (출근~11시)
1. 어제 카드 확인 → "진행 중"이면 자동 "완료"로 닫기
2. 미완료 항목(`[ ]`) 오늘자 카드로 이월
3. 매니저에게 브리핑 + 오늘 할 일 세팅
4. ToBe Daily Report에 오늘자 페이지 생성 (상태: 진행 중)

### 오후 (13시~16시)
1. 오전 세팅 목록 기준 진척도 체크
2. 블로커 감지 (같은 항목 2일 연속 대기 → 패턴 기록)
3. ToBe Daily Report 업데이트

### 퇴근 (17:30~19시)
1. 완료/진행중/대기 상태 최종 확인
2. 매니저 "끝" → ToBe Daily Report 최종 업데이트 (상태: 완료)
3. 완료 항목 `[x]` 체크

## 실시간 업데이트 트리거

| 매니저 발화 | Claude 동작 |
|------------|-------------|
| "이거 끝났어" / "완료" | 해당 항목 `[x]` + 완료건수 +1 |
| "새로운 일 들어왔어" | `[ ]` 항목 추가 + 진행건수 +1 |
| "내일로 미뤄져" / "막혔어" | 대기 상태 변경 + 대기건수 +1 |
| "긴급건" / "ASAP" | ASAP 섹션에 추가 |
| "OO님이랑 같이 하는 거야" | 연관매니저 속성에 추가 |

## 본문 양식

```markdown
## 세일즈 서로 체크 & 공유해야할 내용
- 내용

## ⏰ 당일 or 금주내로 해야 할 업무 (본인 개인)
- [ ] 업무 내용

## 🔆 당일 or 금주내로 해야 할 업무 (업무 요청건)
<당일 해야함 (ASAP)>
- [ ] 업무 내용

<금주내로 해야함>
- [ ] 업무 내용

## 🎯 데일리 체킹 업무
- [ ] 루틴 업무

## 💬 특이사항 / 공유
- 공유 사항
```

## 프로덕트 카드 연동

투두에 `[브랜드명]` 접두사가 있으면 프로젝트 DB에 자동 카드 생성:
- `[키크론] 키크론.kr 리뉴얼` → 프로젝트 DB에 카드 + 데일리에 기록
- 접두사 없으면 → 데일리에만 기록

**우선순위 매핑**: ASAP → P0, 금주 → P1, 기타 → P2

## 주간 리포트

금요일 또는 "이번 주 정리해줘" 시 자동 생성:
- 제목: `{날짜범위} — {이름} — Weekly`
- 유형: Weekly
- 내용: 핵심 수치, 완료 항목, 이월 항목, 다음 주 우선순위

## 비서 행동 원칙

**해야 할 것**: 빠뜨린 루틴 상기, 팔로업, 마감 알림, 블로커 패턴 감지
**하지 말 것**: 추측 업무 추가 금지, 3개 이하 질문, 완료된 항목 재질문 금지, 노션 삭제 금지
