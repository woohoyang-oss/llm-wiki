### 자동화 빌더 상세 스펙

**현재:** 조건/액션이 JSON 텍스트 입력 → 개발자만 사용 가능

**목표:** 시각적 빌더로 누구나 자동화 생성

```
┌──────────────────────────────────────────────────┐
│  자동화 생성                                      │
├──────────────────────────────────────────────────┤
│                                                  │
│  ■ 트리거 (WHEN)                                │
│  ┌────────────────────────────────────────┐      │
│  │ 트리거 유형: [이벤트 기반 ▼]           │      │
│  │                                        │      │
│  │ 이벤트 선택:                           │      │
│  │  ○ 새 메일 수신                        │      │
│  │  ○ 메일 상태 변경                      │      │
│  │  ○ 메일 배정 변경                      │      │
│  │  ○ SLA 임박 (N분 전)                   │      │
│  │  ○ AI 처리 완료                        │      │
│  │  ○ 승인 대기 N시간 초과                │      │
│  │  ○ 고객 재문의 (동일 스레드)           │      │
│  └────────────────────────────────────────┘      │
│                                                  │
│  ■ 조건 (IF)                                    │
│  ┌────────────────────────────────────────┐      │
│  │ [AND ▼]                                │      │
│  │ ┌──────────────────────────────────┐   │      │
│  │ │ [브랜드 ▼] [= ▼] [keychron ▼]   │   │      │
│  │ └──────────────────────────────────┘   │      │
│  │ ┌──────────────────────────────────┐   │      │
│  │ │ [카테고리 ▼] [= ▼] [CS ▼]       │   │      │
│  │ └──────────────────────────────────┘   │      │
│  │ [+ 조건 추가]                          │      │
│  └────────────────────────────────────────┘      │
│                                                  │
│  ■ 액션 (THEN)                                  │
│  ┌────────────────────────────────────────┐      │
│  │ ┌──────────────────────────────────┐   │      │
│  │ │ 1. [에이전트 배정 ▼]             │   │      │
│  │ │    대상: [김민수 ▼]              │   │      │
│  │ └──────────────────────────────────┘   │      │
│  │ ┌──────────────────────────────────┐   │      │
│  │ │ 2. [알림 발송 ▼]                 │   │      │
│  │ │    채널: [Telegram ▼]            │   │      │
│  │ │    메시지: [새 CS 메일: {{제목}}] │   │      │
│  │ └──────────────────────────────────┘   │      │
│  │ ┌──────────────────────────────────┐   │      │
│  │ │ 3. [우선순위 설정 ▼]             │   │      │
│  │ │    값: [high ▼]                  │   │      │
│  │ └──────────────────────────────────┘   │      │
│  │ [+ 액션 추가]                          │      │
│  └────────────────────────────────────────┘      │
│                                                  │
│                         [테스트 실행]  [저장]     │
└──────────────────────────────────────────────────┘
```

**지원 이벤트 트리거:**
| 트리거 | 설명 | 사용 예시 |
|--------|------|-----------|
| `email.received` | 새 메일 수신 | 수신 시 자동 배정 |
| `email.status_changed` | 상태 변경 | solved → 고객 만족도 설문 |
| `email.assigned` | 배정 변경 | 배정 시 알림 |
| `email.sla_warning` | SLA 임박 | 30분 전 알림 |
| `email.ai_processed` | AI 처리 완료 | confidence < 0.5 시 에스컬레이션 |
| `approval.expired` | 승인 대기 초과 | N시간 초과 시 재알림 |
| `email.customer_followup` | 고객 재문의 | 동일 스레드 재문의 시 우선순위 상향 |

**지원 액션:**
| 액션 | 파라미터 | 설명 |
|------|----------|------|
| `assign_agent` | agent_id | 에이전트 배정 |
| `assign_group` | group_id | 그룹 배정 |
| `set_priority` | priority | 우선순위 설정 |
| `set_status` | status | 상태 변경 |
| `add_tag` | tag | 태그 추가 |
| `send_notification` | channel_id, message | 알림 발송 |
| `send_email` | template_id, to | 이메일 발송 |
| `forward` | to_address | 메일 전달 |
| `escalate` | — | 에스컬레이션 |
| `auto_reply` | template_id | 자동 답장 |
| `webhook` | url, payload | 외부 웹훅 호출 |
