## 2. Gmail 패턴 적용 포인트

### 2.1 이메일 리스트 (Gmail → 우리 시스템)

| Gmail 요소 | 우리 시스템 대응 | 구현 위치 |
|-----------|-----------------|-----------|
| 체크박스 (일괄 선택) | ✅ 이미 있음 | emails/page.tsx |
| 별표 (Star) | ★ 중요 표시 / 핀 | priority 필드 활용 |
| 발신자명 (굵게) | 발신자 이름 (읽지 않은 메일은 **bold**) | 리스트 행 |
| 제목 + 미리보기 | 제목 + 첫 줄 미리보기 (회색) | 리스트 행 |
| 라벨 칩 | 태그 배지 (카테고리, 상태) | Badge 컴포넌트 |
| 시간/날짜 | 상대시간 (3시간 전, 어제) | format.ts |
| 읽음/안읽음 | 배경색 구분 (안읽음: 흰색, 읽음: 회색 배경) | 행 스타일 |
| 마우스 오버 액션 | 호버 시 [답장] [전달] [보관] 아이콘 표시 | 행 hover 상태 |

### 2.2 이메일 상세 (Gmail → 우리 시스템)

| Gmail 요소 | 우리 시스템 대응 |
|-----------|-----------------|
| 대화 스레드 | 같은 thread의 메일을 시간순 표시 |
| 답장 입력 | 하단 답장 영역 (리치 텍스트 에디터) |
| 빠른 답장 | AI 생성 초안 버튼 (한 클릭으로 AI 답장 생성) |
| 전달 | 전달 버튼 |
| 라벨 부여 | 태그 추가 드롭다운 |
| 정보 패널 (우측) | 고객 정보 사이드 패널 (customer-info-panel.tsx) |

### 2.3 사이드바 (Gmail → 우리 시스템)

| Gmail 요소 | 우리 시스템 대응 |
|-----------|-----------------|
| 편지쓰기 버튼 | "새 메일" 버튼 (사이드바 상단) |
| 받은편지함 (숫자) | 내 메일함 (미읽음 배지) |
| 라벨 (색상) | 카테고리/태그 (배지 색상) |
| 접기/펼치기 | 그룹별 접기/펼치기 (이미 구현) |
| "기타" 섹션 | 하단 "시스템" 그룹 (admin 전용) |

## 4. 컬러/스타일 시스템

### 4.1 상태 색상 (Gmail 배지와 유사)

| 상태 | 색상 | 사용처 |
|------|------|--------|
| 미읽음/신규 | `blue-500` | 메일 리스트, 배지 |
| 처리중 | `yellow-500` | 상태 배지 |
| 완료 | `green-500` | 상태 배지 |
| 긴급/SLA위반 | `red-500` | 배지, 테두리 강조 |
| AI 자동 | `purple-500` | AI 관련 배지 |
| 개인 | `indigo-500` | scope=personal 표시 |
| 공용 | `gray-500` | scope=shared 표시 |

### 4.2 타이포그래피

- **안읽음 메일**: font-weight: 600 (semibold) — Gmail과 동일
- **읽음 메일**: font-weight: 400 (normal)
- **시간 표시**: 상대시간 사용 (3시간 전, 어제, 3월 10일)
- **미리보기 텍스트**: text-muted-foreground, 한 줄 truncate

### 4.3 반응형 브레이크포인트

| 크기 | 너비 | 레이아웃 |
|------|------|---------|
| Desktop | ≥1280px | 사이드바 + Split Pane (리스트 40% + 상세 60%) |
| Tablet | 768~1279px | 사이드바 접힘 + 전체 너비 리스트 → 클릭 시 상세 페이지 |
| Mobile | <768px | 하단 탭 네비게이션 + 전체 너비 리스트 → 클릭 시 상세 페이지 |

## 5. 핵심 인터랙션 패턴

### 5.1 Gmail에서 가져온 패턴

1. **Split Pane 메일 보기**: 리스트 좌측 + 상세 우측 (Desktop)
2. **대화 스레드**: 같은 thread_id의 메일을 시간순 묶음
3. **호버 액션**: 행에 마우스 오버 시 빠른 액션 아이콘
4. **키보드 단축키**: j/k (탐색), r (답장), e (보관), s (별표), / (검색)
5. **Compose 모달**: 새 메일 작성은 우하단 플로팅 모달 (Gmail의 Compose 창)
6. **라벨/태그**: 색상 배지로 메일 분류 시각화
7. **일괄 작업**: 체크박스 선택 → 상단 툴바로 일괄 처리
8. **알림 벨**: Gmail의 알림과 유사한 우상단 벨 아이콘

### 5.2 우리 시스템 고유 패턴

1. **AI 초안 배너**: 메일 상세 내에 AI 생성 초안을 별도 색상 영역으로 표시
2. **Confidence 미터**: AI 신뢰도를 프로그레스 바로 시각화 (초록/노랑/빨강)
3. **승인 카드**: 승인 대기 메일을 카드 UI로 표시 (승인/수정/거부 버튼 포함)
4. **고객 정보 패널**: 메일 상세 우측에 접기 가능한 고객/주문/배송 정보
5. **지식 저장 모달**: 답변 발송 후 "이 답변을 지식으로 저장?" 팝업
6. **Scope 탭**: 공용/개인 설정을 한 페이지에서 탭으로 전환

## 7. 네비게이션 흐름

### 7.1 에이전트 일반 워크플로우

```
로그인 → Agent 대시보드
    ↓
"긴급 액션" 클릭 → 승인 대기 페이지 (/approvals)
    ↓
승인 카드에서 [승인 발송] 또는 [수정 후 발송]
    ↓
대시보드로 복귀 (또는 다음 승인 건)
    ↓
"내 메일함" 클릭 → /my-inbox (개인 메일)
    ↓
메일 선택 → 우측 상세 → AI 초안 생성 → 발송
    ↓
지식 저장 모달 → [저장] / [건너뛰기]
```

### 7.2 관리자 일반 워크플로우

```
로그인 → Admin 대시보드
    ↓
KPI 확인 → "미처리 보기" 클릭 → 공용 메일함 (/emails)
    ↓
메일 확인 → 배정 / 룰 조정
    ↓
사이드바 "룰 관리" → /rules
    ↓
[공용] 탭 → 룰 추가/수정
    ↓
사이드바 "통계" → /statistics → 기간/에이전트 필터링
```
