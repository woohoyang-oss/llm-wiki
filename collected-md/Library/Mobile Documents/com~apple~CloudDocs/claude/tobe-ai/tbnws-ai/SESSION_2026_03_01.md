**질문**: "EX75 추가발주 필요해?"
**AI 도구 호출**:
1. ✅ Inventory Trend Analysis (keyword: EX75, account: keychron)
2. ✅ Inventory Smart Reorder (action: REORDER)
3. ❌ get_inventory_current(keyword=EX75) 미호출

**AI 응답 요약**:

- 기존 발주: K260123, 2,000개 (4개 모델 × 500개)
- 결론: 추가 발주 불필요 (130일분)

**문제점**:
| # | 문제 | 심각도 |
|---|------|--------|
| 1 | 현재고 수치 미표시 (GT+CJ 합산) | 🔴 HIGH |
| 2 | inventory_current 미호출 → 옵션별 분석 불가 | 🔴 HIGH |
| 3 | 2,000÷15.4=130일 (전체 합산만) — 인기 옵션은 50일일 수 있음 | 🟡 MED |
| 4 | 옵션별 소진일 미계산 | 🟡 MED |

**ROOT CAUSE #1 (commit 5e08504)**:

**ROOT CAUSE #2 (commit 4f6a8cb)**:

- 수정: keyword/brand 파라미터 추가, ORDER BY 앞에 조건 삽입

**ROOT CAUSE #3 — AI가 3개 도구 호출을 꺼림**:
- 프롬프트에서 inventory_current 필수 명시했지만, AI가 2개 도구만 호출하고 결론 내림
- **해결 (commit 96f27bd)**: trend_analysis API에 `option_stocks` 자동 포함
  - keyword 기반으로 CURRENT_STOCK 조회하여 옵션별 재고 데이터 포함
  - AI가 trend_analysis 하나만 호출해도 현재고 + 옵션별 분석 가능
  - `OptionStock` 모델: product_code, option_name, total_stock, monthly_sales, sales_share_pct, days_of_supply

**검증된 데이터 (API 직접 확인)**:
