# ToBe Networks AI 분석 시스템 아키텍처 계획

> 최종 업데이트: 2026-02-28
> 목표: 투비네트웍스의 재무/회계/재고/발주/세일즈/마케팅/수출입 전 데이터를 파악하고,
> 과거를 분석해 미래를 예측하며 각종 인사이트를 제안하는 AI 시스템

---

## 1. 시스템 개요

### 핵심 목표
- 9개 MySQL DB + 1개 PostgreSQL의 **229개 테이블**을 정확히 이해하는 AI
- 자연어 질문 → SQL 생성 → 정확한 답변 (Text-to-SQL)
- 역할별 전문 에이전트 (재무, 회계, 세일즈, 마케팅, 수출입, 재고, HR)
- 과거 데이터 분석 → 미래 예측 → 액션 제안

## 2. 전문 에이전트 설계

### 2.1 세일즈 에이전트 (Sales Agent)

**담당 영역**: 매출 분석, 채널별 성과, 목표 달성률, 판매 추이

**접근 테이블**:
- `tbnws_sabangnet_order` - 사방넷 온라인 주문
- `erp_coupang_order` - 쿠팡 주문
- `naver_commerce` - 네이버 주문
- `erp_order_recv` + `erp_order_recv_item` - B2B 수주
- `erp_annual_sales` - 연간 매출
- `erp_annual_model_sales` - 모델별 매출
- `erp_target_amount` - 매출 목표
- `erp_customer_month_targert_sales` - 월별 고객 매출 목표
- `erp_channel_list` + `erp_channel_price` - 채널별 가격
- `erp_partner` - 거래처

**질문 예시**:
- "이번 달 키크론 매출이 목표 대비 얼마나 달성했어?"
- "온라인 채널 중 가장 매출이 높은 곳은?"
- "전년 동기 대비 매출 증감률은?"
- "브랜드스토어 vs 스마트스토어 AOV 비교"
- "B2B 거래처 중 매출 하락한 곳은?"

**인사이트 자동 생성**:
- 채널별 매출 비중 변화 감지
- 목표 미달 브랜드 경고
- 전월/전년 대비 이상 매출 패턴 감지
- 시즌별 판매 패턴 분석

---

### 2.2 마케팅 에이전트 (Marketing Agent)

**담당 영역**: 광고 성과, ROAS, 키워드 효율, 캠페인 분석

**접근 데이터 소스**:
- `naver_search_ad` - SA 검색광고
- `naver_ranking_tracker_list` - 순위 추적
- `naver_shopping_ranking` - 쇼핑 랭킹
- `naver_store_review_crawler` - 리뷰 데이터
- tbe.kr API 데이터 (SA/GFA 통합)

**질문 예시**:
- "이번 주 ROAS가 가장 높은 키워드 TOP 5는?"
- "비효율 키워드 중 즉시 OFF 해야 할 것은?"
- "GFA 캠페인 중 전환율 가장 높은 건?"
- "광고비 대비 실제 스토어 매출 비율은?"
- "경쟁사 대비 네이버 순위 변화는?"

**인사이트 자동 생성**:
- 비효율 키워드 자동 감지 & OFF 추천
- ROAS 급락 알림
- 비즈머니 잔액 소진 예측
- 전환율 변화 트렌드
- 광고비 대비 실제매출 GAP 분석

---

### 2.3 재고/발주 에이전트 (Inventory Agent)

**담당 영역**: 재고 현황, 발주 추천, 품절 예측, 과잉재고 감지

**접근 테이블**:
- `erp_stock` + `erp_stock_summary` - 재고
- `erp_stock_adjustment` - 재고 조정
- `erp_order` + `erp_order_item` - 발주
- `erp_order_payment_history` - 결제 이력
- `erp_export_schedule` + `erp_export_schedule_item` - 출고 스케줄
- `erp_product_info` - 상품 마스터
- `erp_refurb_stock` - 리퍼브 재고
- `wms_relocation` - 창고 이동

**질문 예시**:
- "지금 긴급 발주가 필요한 상품은?"
- "K10 Pro SE 재고가 언제 소진될 것 같아?"
- "과잉재고로 자금이 묶인 금액은?"
- "이번 달 입고 예정 수량은?"
- "리퍼브 재고 중 판매 가능한 건?"

**인사이트 자동 생성**:
- 가중평균 판매 기반 소진일 예측
- 놓쳐진 발주 감지 (재고 4주 미만 + 발주 없음)
- 소진 속도 초과 경고 (3달 평균 대비 1.5배)
- 체류 재고 리포트 (사장/저회전/확인필요)
- 리드타임 기반 발주 타이밍 추천

---

### 2.4 재무/회계 에이전트 (Finance Agent)

**담당 영역**: 세금계산서, 견적서, 경비, 매입/매출, 환율

**접근 테이블**:
- `finance_tax_invoice` + `finance_tax_invoice_detail` - 세금계산서
- `finance_estimate` + `finance_estimate_item` - 견적서
- `finance_etc_remit` - 기타 경비/송금
- `finance_fixtures_remit` - 구매 요청
- `currency_exchange_rate` - 환율
- `erp_order_payment_history` - 결제 이력
- `erp_order_recv` - B2B 수주 (미수금 관련)

**질문 예시**:
- "이번 달 미수금 현황은?"
- "세금계산서 미발행 건이 있어?"
- "경비 지출 TOP 5 항목은?"
- "전월 대비 매입원가 변동은?"
- "환율 변동이 매입가에 미치는 영향은?"

**인사이트 자동 생성**:
- 미수금 기한 초과 경고
- 세금계산서 미발행 알림
- 월별/분기별 경비 추이
- 매입원가 변동 → 마진율 영향 분석
- 환율 변동 리스크 알림

**질문 예시**:
- "현재 운송 중인 발주 건은?"
- "이번 주 입고 예정인 컨테이너는?"
- "리드타임이 가장 긴 공급처는?"
- "통관 지연 중인 건이 있어?"
- "올해 해외 발주 총액은?"

**인사이트 자동 생성**:
- 입고 지연 알림
- 리드타임 추이 분석
- 공급처별 납기 준수율

## 3. 기술 스택 선정

### 3.1 LLM 선택
| 옵션 | 장점 | 단점 |
|------|------|------|
| **Claude (Anthropic)** | 긴 컨텍스트, 정확한 SQL, 한국어 우수 | 비용 |
| GPT-4o/5 | 범용성, Tool Use 안정성 | - |
| **자체 파인튜닝** | 도메인 특화, 비용 절감 | 초기 투자 |

**추천**: Claude API (sonnet/haiku) + 자체 프롬프트 엔지니어링
- 단순 질의: Haiku (빠르고 저렴)
- 복합 분석: Sonnet (정확하고 깊이)

### 3.2 핵심 기술 컴포넌트

```
┌─────────────────────────────────────────────────┐
│  1. Schema Context Manager                      │
│  - 229개 테이블의 스키마를 구조화된 문서로 관리        │
│  - 에이전트별 관련 테이블만 선택적 주입               │
│  - 테이블 간 관계(FK) 명시                          │
└─────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────┐
│  2. Text-to-SQL Engine                          │
│  - 자연어 → SQL 변환                               │
│  - 읽기 전용 (SELECT만 허용)                        │
│  - 쿼리 검증 & 최적화 레이어                         │
│  - 멀티 DB 라우팅 (9개 MySQL + 1 PG)              │
└─────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────┐
│  3. RAG (Retrieval-Augmented Generation)        │
│  - 비즈니스 규칙/정책 문서 벡터 검색                  │
│  - 과거 질문/답변 캐시                               │
│  - 상품 정보, 파트너 정보 검색                        │
│  - PostgreSQL pgvector 활용                       │
└─────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────┐
│  4. Prediction Engine                           │
│  - 시계열 예측 (판매량, 재고 소진)                    │
│  - 이상 감지 (매출 급변, 재고 이상)                   │
│  - 트렌드 분석 (YoY, MoM, WoW)                    │
└─────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────┐
│  5. Action Recommender                          │
│  - 분석 결과 → 구체적 액션 제안                      │
│  - 발주 추천, 키워드 ON/OFF, 가격 조정               │
│  - 우선순위 기반 태스크 리스트 생성                    │
└─────────────────────────────────────────────────┘
```

### 3.3 서버 아키텍처

```
[NestJS API Server] ← 기존 tbnws_agent_server 확장
  ├── /api/chat              → 채팅 인터페이스
  ├── /api/agent/:role       → 역할별 에이전트 호출
  ├── /api/query             → Text-to-SQL 실행
  ├── /api/insight/:domain   → 도메인별 인사이트
  └── /api/predict/:target   → 예측 요청

[데이터베이스 접근]
  ├── Read Replica (읽기 전용 복제본 권장)
  ├── Connection Pool (에이전트당 제한)
  └── Query Timeout (30초)

[캐시 레이어]
  ├── Redis → 자주 묻는 질문 캐시
  ├── Redis → 일일 KPI 스냅샷
  └── Redis → 세션 컨텍스트
```

## 4. 데이터 파이프라인

### 4.1 실시간 데이터
- **사방넷 주문**: 크론 5분 간격 동기화
- **네이버 주문**: API 실시간 연동
- **쿠팡 주문**: API 실시간 연동
- **재고 변동**: erp_stock INSERT/UPDATE 트리거

### 4.2 일일 배치
- **일일 KPI 스냅샷**: 매일 00:00 집계
  - 일 매출 합산, 채널별 매출, 브랜드별 매출
  - 재고 현황 스냅샷
  - 발주 진행 현황
- **판매 속도 계산**: 가중평균 판매량 (1주/1개월/3개월)
- **이상 감지**: 전일/전주/전월 대비 편차 계산

### 4.3 주간/월간 배치
- **매출 목표 달성률** 계산
- **재고 회전율** 계산
- **체류 재고** 분류 (사장/저회전/확인필요)
- **공급처 납기 준수율** 집계

## 5. 프롬프트 엔지니어링

### 5.1 스키마 컨텍스트 전략

**문제**: 229개 테이블 전체를 프롬프트에 넣으면 컨텍스트 낭비
**해결**: 에이전트별 관련 테이블만 선택적 주입

```
[Router Agent]
→ 질문 분류 (세일즈/재고/재무/마케팅/HR/수출입)
→ 해당 에이전트 호출 시 관련 스키마만 첨부

예: "키크론 이번 달 매출" → Sales Agent
    → erp_annual_sales, tbnws_sabangnet_order,
      erp_product_info, erp_partner 스키마만 주입
```

### 5.2 시스템 프롬프트 구조

```markdown
## 역할
당신은 투비네트웍스의 {역할명} 전문 AI 어시스턴트입니다.

## 데이터베이스 스키마
{해당 도메인 테이블 DDL}

## 테이블 관계
{FK 관계 설명}

## 비즈니스 규칙
- product_code는 "카테고리-브랜드-상품-옵션" 형식 (예: G-0029-0405-0064)
- sell_status: Y=판매중, N=미판매, D=삭제
- partner_type: IMPORT=수입, EXPORT=출고, BOTH=양방향
- ON 채널: 자사 직접 판매 (브랜드스토어, 스마트스토어 등)
- OFF 채널: 도매/B2B (쿠팡 로켓, 밸류포인트 등)

## 응답 규칙
1. 데이터가 필요하면 SQL 쿼리를 생성하세요 (SELECT만 가능)
2. 숫자는 한국 원화 단위로 표시 (억/천만/만)
3. 비교 분석 시 전월/전년 동기 대비를 함께 제시
4. 이상 패턴 발견 시 원인 추정과 액션 제안을 포함
5. 불확실한 정보는 명시적으로 "추정"임을 표기
```

### 5.3 Few-shot 예시

```
Q: "키크론 이번 달 매출은?"
SQL: SELECT SUM(pay_cost) as total_sales, COUNT(*) as order_count
     FROM tbnws_sabangnet_order
     WHERE model_no LIKE 'G-0029%'
     AND order_date BETWEEN '2026-02-01' AND '2026-02-28'
     AND order_status NOT IN ('취소')
```

## 7. 보안 & 접근 제어

### 데이터 접근 원칙
1. **읽기 전용**: AI는 절대 데이터를 수정/삭제하지 않음
2. **Read Replica**: 프로덕션 DB에 직접 접근하지 않음
3. **쿼리 제한**: SELECT만 허용, LIMIT 강제, 타임아웃 30초
4. **권한 기반**: 사용자 role에 따라 접근 가능한 에이전트 제한
5. **로깅**: 모든 AI 쿼리 & 응답 기록 (`tbnws_admin_log`)

### 민감 데이터 보호
- 계좌번호, 사업자번호 등 마스킹 처리
- 급여/인사 정보는 HR 권한자만 접근
- 개인정보(이름, 전화번호) 응답 시 부분 마스킹
