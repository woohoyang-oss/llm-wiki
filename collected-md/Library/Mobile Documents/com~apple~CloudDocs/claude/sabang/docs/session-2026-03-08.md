def _migrate_source_name_column(self):
    """마이그레이션: batch_configs에 source_name 컬럼 추가"""
    try:
        self.conn.execute("SELECT source_name FROM batch_configs LIMIT 1")
    except Exception:
        self.conn.execute("ALTER TABLE batch_configs ADD COLUMN source_name TEXT DEFAULT ''")

### SCP 옵션 이슈
- `-o PubkeyAuthentication=no -o PreferredAuthentications=password` 추가 시 **Permission denied** 발생
- 옵션 제거하고 `-o StrictHostKeyChecking=no`만 사용 시 정상 동작
- 원인 추정: 서버가 pubkey 인증을 우선 시도하되, password도 허용하는 설정. PubkeyAuthentication=no를 강제하면 ssh-agent의 키가 무시되면서 비밀번호 인증도 실패하는 것으로 추정

## 트러블슈팅 기록

### 1. 행 펼침 동작 안 함
- **증상**: 이력 조회에서 행 클릭 시 아무 반응 없음
- **디버깅**: `toggleHistoryDetail` 검색 → 2곳 정의 발견 (line 2408, line 2998)
- **원인**: JS는 같은 이름의 함수를 나중에 정의하면 덮어씀
- **수정**: 두 번째를 `toggleBatchHistoryDetail`로 rename

### 2. 상세 테이블 컬럼 잘림 ("F...", "S...", "0...")
- **증상**: G-코드별 상세 테이블에서 모든 텍스트가 잘려서 "F...", "S..." 등으로 표시
- **원인**: `products-table` CSS의 `table-layout: fixed` + `overflow: hidden; text-overflow: ellipsis; white-space: nowrap`
- **수정**: `products-table` 클래스 대신 inline 스타일로 `table-layout: auto` 적용

### 3. "▶ ▶ G-코드별 상세" 중복 삼각형
- **증상**: `<details><summary>` 앞에 브라우저 네이티브 ▶ + 텍스트 ▶ = 두 개
- **원인**: `_buildResultDetailTable`에서 summary 텍스트에 `▶` 문자 포함
- **수정**: 텍스트에서 `▶ ` 제거 → 네이티브 disclosure triangle만 표시

### 4. SCP Permission denied
- **증상**: `sshpass -p 'ec2-user' scp -o PubkeyAuthentication=no -o PreferredAuthentications=password ...` 실패
- **수정**: `-o PubkeyAuthentication=no -o PreferredAuthentications=password` 옵션 제거
- **CLAUDE.md 반영**: 배포 커맨드 업데이트
