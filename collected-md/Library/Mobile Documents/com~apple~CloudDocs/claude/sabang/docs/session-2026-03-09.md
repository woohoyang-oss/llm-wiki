## 1. 쇼핑몰 송신 on/off 토글 추가

### 배경
- 삼성카드쇼핑 등 수정 불가능한 쇼핑몰은 송신해도 소용없음
- 쇼핑몰 설정 탭에서 개별 쇼핑몰을 송신 제외할 수 있는 토글 필요

### 변경 내용

#### `dashboard/db.py` — enabled 컬럼 추가
- `_migrate_mall_enabled_column()`: `mall_code_mappings` 테이블에 `enabled INTEGER NOT NULL DEFAULT 1` 컬럼 마이그레이션
- `get_mall_code_mappings()`: enabled 필드 포함 반환
- `update_mall_code_mapping()`: code_type과 enabled를 독립적으로 업데이트 가능하도록 리팩터링
- `get_mall_code_map()`: enabled=0인 쇼핑몰은 `'disabled'` 반환

#### `dashboard/app.py` — PUT API 확장
- `PUT /api/mall-mappings/<shma_id>`: `{"enabled": 0}` 또는 `{"enabled": 1}` 처리 추가
- code_type과 enabled 동시 또는 개별 업데이트 지원

#### `dashboard/templates/index.html` — 토글 UI
- `renderMallCard()`: 몰 이름 왼쪽에 CSS 토글 스위치 추가
- off 시: 카드 dim (opacity 0.45) + 코드 타입 버튼 비활성화 (`pointer-events: none`) + "송신제외" 뱃지
- `toggleMallEnabled(shmaId, enabled)`: PUT API 호출 → 목록 새로고침
- `loadMallMappings()`: 송신제외 카운트 표시

#### `skills/soldout.py` — disabled 쇼핑몰 필터링
- `sabang_build_post_data()`: `code_type == 'disabled'`인 trList 항목 skip
- 로그 출력: `[FILTER] 송신제외 N건 (disabled 쇼핑몰)`

### 동작 흐름
1. 쇼핑몰 설정 탭에서 토글 off → DB `enabled=0` 저장
2. 배치/수동 실행 시 `get_mall_code_map()` → disabled 쇼핑몰은 `'disabled'` 반환
3. `sabang_build_post_data()`에서 해당 쇼핑몰 trList 항목 자동 제외
