> **작성일:** 2026-03-06
> **최종수정:** 2026-03-06 (영상 분석 + 스크래퍼 테스트 반영)
> **업무 개요:** 구글 스프레드시트의 재고 데이터를 기반으로, 사방넷에서 재고 부족 SKU를 찾아 품절 처리하는 프로세스

## 1. 업무 목적

재고가 부족하거나 소진된 상품(SKU)을 구글 스프레드시트에서 확인한 뒤, 사방넷(Sabangnet) 쇼핑몰 통합관리 시스템을 통해 연동된 모든 판매 채널(카페ON, 삼성카드, 쿠팡 등)에 일괄 품절 처리를 송신하는 업무입니다.

## 3. 상세 절차

### STEP 1. 구글 스프레드시트에서 품절 대상 SKU 확인

1. 구글 스프레드시트 **"현대이지웰_마스터시트"**를 엽니다.
2. 시트에서 **재고 수량이 부족한 상품**을 확인합니다.
   - 분홍색(핑크) 하이라이트된 행이 품절/재고부족 상품입니다.
3. 해당 상품의 **자체상품코드(G-코드)**를 메모합니다.
   - 예시: `G-0029-0479-0002`
   - G-코드는 스프레드시트의 좌측 열에 표기되어 있습니다.

> **시트 구성:** 투비코드, 대분류, 상품명, 옵션, 재고수량, 승인단가기준, 이지몰 판매가 등의 컬럼. 하단 탭에 브랜드/카테고리별 시트 분리:
> - 260226예비용 / **키크론** (주력) / 지티기어 / 페퍼톡_강아지사료 / 가습기 / Xirfy / 이지웰_raw / 투비재고_v2 등

**자동화 방법:** `gog sheets` CLI로 스프레드시트 데이터 읽기
```bash
gog sheets get 1ItUgaLRuxnQWN7hZRS_SjaZSbZ9WEEnshgkZQEMAXvU --range "시트명!A:Z" --account ai.login@tbnws.com
```

### STEP 2. 사방넷 접속 및 대시보드 확인

1. `www.sabangnet.co.kr/login/login-main`에서 로그인합니다.
2. 로그인 후 "사방넷 접속" 버튼 클릭 → `sbadmin13.sabangnet.co.kr` 관리자 페이지 진입
3. **Dashboard**에서 현재 상품 현황을 확인합니다.
   - 주문현황: 주문 건수, 상품현황 건수, 상품미확인 건수, 미전달주문 건수
   - 상품 현황: 전체상품 수, 공급중, 일시정지, 완전품절
   - C/S 현황, 결제대 현황 등

**자동화 방법:** Playwright headless 로그인 (검증 완료)
```python
from playwright.async_api import async_playwright

async with async_playwright() as p:
    browser = await p.chromium.launch(headless=True)
    page = await browser.new_page()
    await page.goto('https://www.sabangnet.co.kr/login/login-main')
    await page.wait_for_selector('input[type="text"]')
    await page.fill('input[type="text"][placeholder*="아이디"]', 'ai_tobe')
    await page.fill('input[type="password"]', '<비밀번호>')
    await page.click('button:has-text("로그인")')
    # 로그인 성공 시 쿠키 38개 발급, URL -> www.sabangnet.co.kr/ 리다이렉트
```

### STEP 3. 쇼핑몰상품수정 메뉴 진입

1. 좌측 메뉴에서 **쇼핑몰관리** > **쇼핑몰상품수정**을 클릭합니다.
2. "쇼핑몰상품수정(또는 판매기간 연장처리)" 페이지가 열립니다.

---

### STEP 4. 검색 조건 설정

1. **검색항목** 드롭다운을 클릭합니다.
2. 드롭다운 목록에서 **"자체상품코드"**를 선택합니다.
   - 드롭다운 옵션: 상품명(송신), 상품명(사방넷), 상품메모, 쇼핑몰상품코드, **자체상품코드**, 송신No, 부가정보코드(옵션)
3. 검색창에 **STEP 1에서 메모한 G-코드**를 입력합니다.
   - 예: `G-0029-0479-0002`
4. **검색** 버튼을 클릭합니다.

---

### STEP 5. 검색 결과 확인 및 상품 선택

1. 검색 결과로 해당 G-코드에 매칭되는 상품 목록이 표시됩니다.
   - 하나의 G-코드가 여러 쇼핑몰에 등록 → **다수의 결과** (예: 9건)
2. 결과 테이블 컬럼:
   - No, 송신구분(Manage), 쇼핑몰/쇼핑몰ID, 최종송신일시, 쇼핑몰상품코드, 자체상품코드/자체옵션코드, 상품명(송신), 송신가격/매체가격(판매가), 배송비구분, 판매가(송신), 모델명/브랜드 등
3. 품절 처리할 상품들의 **체크박스를 선택**합니다.
   - 노란색 행 = 에러/변환 불가 항목 → 건너뜀

> **주의:** "상품선택 변환 실패" 메시지가 나오는 항목은 이미 품절 상태이거나 상태 변환 조건이 맞지 않는 경우입니다.

---

### STEP 6. 상품수정 송신 실행

1. 상품 선택 후, **"상품수정 송신"** 버튼을 클릭합니다.
2. **"상품수정 송신" 확인 팝업(모달)**이 열립니다.
   - 판매상태와 옵션재고/수량 관련 항목만 수정됨
   - 선택항목에 의한 수정 범위 안내 표시
   - 수정 원칙과 범위에 대한 안내사항
3. 확인 후 **"송신"** 버튼을 클릭합니다.

---

### STEP 7. 품절 송신 진행 및 완료 확인

> **[수정] 영상 분석 결과:** 품절 송신은 별도 로컬 클라이언트(`127.0.0.1:8181`)가 아닌, **사방넷 웹 관리자 페이지 내 팝업(모달)에서 직접 실행**됩니다.

1. 송신 버튼 클릭 후 **사방넷 로고가 있는 로딩 팝업**이 표시됩니다.
   - 파란색 사방넷 로고 + 로딩 애니메이션
2. 로딩 완료 후 **빨간 배경의 송신 결과 팝업**이 나타납니다.
   - 실시간으로 송신 진행 로그가 표시됨
   - "N번 품절정보를 송신중입니다" → "송신성공" 등의 메시지
   - 각 쇼핑몰별 송신 성공/실패 결과 확인 가능
3. 모든 건의 송신이 완료될 때까지 대기합니다.

> **주의사항:**
> - 송신 도중 팝업 창이나 브라우저를 닫지 마세요.
> - 1회 작업시 500건 이하 권장.
> - 브라우저 자동화 시 팝업 완료 대기 로직 필요.

## 6. 주의사항 및 팁

1. **G-코드 정확성:** 스프레드시트에서 복사한 G-코드가 정확한지 확인. 한 글자라도 다르면 검색 불가.
2. **다중 채널 확인:** 하나의 상품이 여러 쇼핑몰에 등록 → 검색 결과의 모든 채널을 선택해야 전 채널 품절 처리.
3. **노란 행 건너뛰기:** 결과에서 노란색 하이라이트 행은 변환 불가 항목 → 체크하지 않음.
4. **작업 시간대:** 쇼핑몰 시스템 점검 시간 피해 작업.
5. **송신 건수:** 1회 500건 이하 권장.

## 7. 문제 발생 시 대응

| 상황 | 대응 방법 |
|------|-----------|
| G-코드 검색 결과 없음 | 코드 오탈자 확인, 스프레드시트와 사방넷의 코드 체계 일치 여부 확인 |
| 로그인 실패 | 사방넷 SPA 로딩 대기 시간 조정 (wait_for_selector 타임아웃 늘리기) |
| 송신 팝업 로딩 무한 | 인터넷 연결 확인 후 재시도, 브라우저 새로고침 |
| 송신 중 오류 발생 | 500건 이하로 분할 송신, 인터넷 연결 확인 |
| 특정 쇼핑몰만 실패 | 해당 쇼핑몰 시스템 상태 확인 후 개별 재송신 |
| Playwright 셀렉터 실패 | sbadmin13 페이지 구조 변경 가능성 → 셀렉터 재분석 필요 |
