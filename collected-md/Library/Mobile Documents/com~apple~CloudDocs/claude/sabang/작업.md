match = re.search(r'자료수\s+(\d+)\s+건', count_text)

table_rows = page.locator('.el-table__body-wrapper .el-table__row')
for i in range(await table_rows.count()):
    row = table_rows.nth(i)
    bg = await row.evaluate('el => getComputedStyle(el).backgroundColor')
    if 'yellow' in bg.lower() or 'rgb(255, 255' in bg:

## 7. 테스트 결과 로그

### 테스트 2-1: Playwright 로그인 + admin 이동 + 메뉴 진입 ✅
- 로그인 성공 → sbadmin13 진입 → 쇼핑몰상품수정 페이지 로드
- 모두 정상 작동

### 테스트 2-2: 자체상품코드 드롭다운 선택 ❌
- 1차 시도: strict mode violation (2개 매칭) → `.first` 추가
- 2차 시도: **Timeout - element is not visible**
  - el-select dropdown item을 찾았으나 (resolved) 화면에 보이지 않음
  - headless 모드에서 el-select 드롭다운 팝업이 display:none 상태로 유지
  - **원인 추정:** Element UI el-select는 클릭 시 `el-select-dropdown` 팝업을 body 끝에 append하는데, headless에서 렌더링 타이밍 문제
  - **해결 방안 후보:**
    1. `page.evaluate()`로 Vue 컴포넌트 내부 값을 직접 변경
    2. 드롭다운 팝업의 display를 강제로 변경 후 클릭
    3. `force: True` 옵션으로 visibility 무시하고 클릭
    4. headless=False로 테스트하여 렌더링 확인

### 테스트 2-3: Playwright 검증 (test_verify.py) ⚠️
- 2개 랜덤 상품 선택: G-0029-0396-0001 (키크론 B Pro), G-0029-0167-0010 (액세서리 케이블)
- 스프레드시트 데이터:
  - A: 재고=0, 등급=A, 구상품코드=1034666643 ✅
  - B: 재고=0, 등급=X, 구상품코드="등록필요" ⚠️
- el-select 3가지 방법 시도: (1) Vue `__vue__.$children` → empty array, (2) `force=True` → 실패, (3) display 강제변경 → 클릭은 성공하나 값 변경 안 됨
- 결과: 두 제품 모두 9건 반환 BUT 드롭다운 값이 '' (빈값) → **기본 검색 실행됨** (자체상품코드 아님)

### 테스트 3-1: Scrapling StealthyFetcher (test_scrapling.py) ⚠️
- **엔진:** Camoufox (Firefox headless)

from scrapling import StealthyFetcher

def page_action(page):

    # 2. JavaScript로 옵션 직접 클릭 (Vue reactive state 트리거)
    page.evaluate('''() => {
        const dropdowns = document.querySelectorAll('.el-select-dropdown');
        for (const d of dropdowns) {
            const items = d.querySelectorAll('.el-select-dropdown__item');
            if (items.length >= 15) {  // 검색항목 드롭다운 (18개)
                for (const item of items) {
                    if (item.textContent.trim() === '자체상품코드') {
                        d.style.display = 'block';
                        d.style.visibility = 'visible';
                        item.click();
                        return;
                    }
                }
            }
        }
    }''')
    page.wait_for_timeout(500)
    return page

response = StealthyFetcher.fetch(
    url,
    headless=True,
    page_action=page_action,
    humanize=False,
    google_search=False,

### 방법별 결과 비교

| 방법 | 엔진 | 기술 | 결과 |
|------|------|------|------|
| Playwright locator click | Chromium | `.click()` | ❌ display:none, not visible |
| Playwright force click | Chromium | `.click(force=True)` | ❌ 클릭되나 값 변경 안 됨 |
| Vue `__vue__` 직접 접근 | Chromium | `el.__vue__.$children` | ❌ $children 비어있음 |
| display 강제 + force click | Chromium | CSS + `.click(force=True)` | ❌ 값 변경 안 됨 |
| dispatchEvent | Camoufox | `nativeInputValueSetter` | ⚠️ 표시값만 변경 (Vue state 미변경) |
| **JS element.click()** | **Camoufox** | **`item.click()`** | **✅ Vue state 정상 변경** |

## 9. 주의사항 / 트러블슈팅

1. **탭 Detach 문제:** Chrome MCP에서 sbadmin13 탭이 자주 detach됨 → SPA 내부 라우팅/팝업 때문으로 추정
2. **el-select는 <select> 아님:** `select_option()` 사용 불가, 반드시 클릭→옵션클릭 방식
3. **자체상품코드 2개 매칭:** 검색항목 드롭다운(18개)과 정렬기준 드롭다운(12개)에 각각 존재 → items.length >= 15로 구분
4. **SPA 로딩:** 페이지 이동 후 wait_for_selector 필요 (Vue 비동기 렌더링)
5. **팝업 간섭:** 대시보드 진입 시 광고/공지 팝업이 나타남 → 메뉴 클릭 대신 직접 URL 이동이 안정적
6. **Camoufox Firefox 제한:** `__vue__` 접근 불가 (Vue devtools 훅 없음) → JS element.click()으로 우회
7. **Camoufox macOS:** `headless='virtual'` 사용 불가 (Linux Xvfb 전용) → `headless=True` 사용
8. **Camoufox 설치 필요:** 최초 1회 `python3 -m camoufox fetch` 실행 (304MB Firefox 다운로드)
9. **스프레드시트 데이터 품질:** 등급 X / 구상품코드 "등록필요" 상품은 사방넷 미등록 → 자동화 시 스킵 필요
