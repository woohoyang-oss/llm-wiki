# CS 미답변 및 클레임 관리

## 1단계: 데이터 수집 (병렬 실행)

다음 MCP 도구를 병렬로 호출하세요:

- `get_cs_overview` — CS 전체 현황
- `get_unanswered_inquiries` — 미답변 문의
- `get_qna_unanswered` — QnA 미답변
- `get_inquiry_answer_rate` — 문의 답변율
- `get_claims_summary` — 클레임 요약
- `get_claims_rate` — 클레임 발생률

## 2단계: 분류 및 우선순위 지정

수집된 문의를 아래 기준으로 분류하세요:

### 우선순위 기준
1. **긴급** — 클레임, 환불/교환 요청, 배송사고
2. **높음** — 상품 문의(구매 전), 불만 접수
3. **보통** — 일반 문의, 배송 조회
4. **낮음** — 단순 확인, 중복 문의

### 리포트 형식

```
📞 CS 미답변 현황 (YYYY-MM-DD)

■ 요약
- 총 미답변: X건 (문의 X건 + QnA X건)
- 답변율: X% (목표: 95%)
- 클레임: X건 (처리율 X%)

■ 긴급 대응 필요 🚨
1. [고객명] - [문의 내용 요약] - [접수일시]
2. ...

■ 미답변 목록 (우선순위순)
| # | 우선순위 | 유형 | 내용 요약 | 접수일 | 경과시간 |
|---|---------|------|----------|--------|---------|
| 1 | 긴급    | 클레임 | ...     | ...    | ...     |
| 2 | 높음    | 상품문의 | ...   | ...    | ...     |

■ 답변 초안
[각 미답변 건에 대해 답변 초안 제공]
- 문의 #1: "고객님, ..."
- 문의 #2: "안녕하세요, ..."
```

## 3단계: 답변 초안 작성 원칙

- 정중하고 공감하는 톤 유지
- 구체적인 해결 방안 또는 다음 단계 안내
- 배송 관련은 택배사/송장번호 포함
- 환불/교환은 절차와 예상 소요기간 안내
- 불확실한 내용은 "확인 후 안내드리겠습니다"로 처리
