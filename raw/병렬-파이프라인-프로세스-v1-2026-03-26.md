## 병렬 파이프라인 프로세스 (필수)
모든 태스크는 아래 병렬 파이프라인을 따릅니다. **순차 진행 금지.**

### Phase 1: 병렬 사전 분석 (동시 실행)
태스크를 받으면 다음 3개를 **동시에** Agent로 실행:

| Agent | 역할 | 산출물 |
|-------|------|--------|
| **Explore (분석)** | 영향 범위 파악, 관련 파일/함수 식별, 아키텍처 리스크 | `분석 보고서` |
| **Reviewer (사전 리뷰)** | 관련 코드 현재 품질, 기존 버그/기술부채, 주의사항 | `사전 리뷰 보고서` |
| **Tester (테스트 설계)** | 기존 테스트 커버리지 확인, 테스트 케이스 선설계 | `테스트 케이스 명세` |

### Phase 2: 구현 명세 확정
Phase 1 결과를 종합하여 **Plan agent**로 구현 명세 확정.
- 분석 보고서의 영향 범위 반영
- 사전 리뷰의 주의사항 반영
- 테스트 케이스와 구현이 정합하는지 확인

### Phase 3: 구현 + 테스트 코드 (동시 실행)
| Agent | 역할 |
|-------|------|
| **Coder** | 구현 명세에 따라 코드 작성 (worktree 격리) |
| **Tester** | Phase 1 테스트 명세 기반으로 테스트 코드 작성 (worktree 격리) |

### Phase 4: 검증 (동시 실행)
| Agent | 역할 |
|-------|------|
| **Reviewer** | 구현 코드 리뷰 (품질, 보안, CLAUDE.md 규칙 준수) |
| **Tester** | 테스트 실행 + 결과 보고 |

### Phase 5: 수정 및 완료
- 리뷰/테스트 피드백 반영 → Coder 수정
- 최종 테스트 통과 확인 → 머지

### 프로세스 준수 규칙
1. **Phase 1은 반드시 3개 Agent를 동시 실행**해야 함 (순차 금지)
2. **Phase 3은 반드시 Coder + Tester 동시 실행**해야 함
3. **Phase 4는 반드시 Reviewer + Tester 동시 실행**해야 함
4. 각 Phase 완료 시 사용자에게 요약 보고
5. 단순 수정(오타, 설정 변경 등)은 Phase 1만 스킵 가능 (Phase 3~4는 필수)
