# Claude Team Bible v1.0

팀형 Claude 에이전트 구축 및 운영 바이블.
새 프로젝트에 동일 구조를 빠르게 복제하기 위한 참조 매뉴얼.

---

## 1. 팀 구성 방식
- 6+1 구조: TL, PL, Coder, Reviewer, Tester(Slack), Tester(아카샤), 커맨드센터
- 하드웨어 배치: 맥북(TL+PL), 맥미니(Coder+Reviewer+Tester), 맥스튜디오(커맨드센터+Tester)

## 2. 통신 방식
- Slack (실시간 지시/보고)
- 아카샤 (비동기 태스크/기록)

## 3. 대시보드/모니터링
- Slack 채널 = 실시간 대시보드
- 아카샤 = 영구 기록

## 4. 리소스 분산 방식
- 작업 유형별 배분
- 같은 파일 충돌 방지
- 5분 타임아웃 대응

## 5. 메시지 전파 방식
- 하향 전파, 상향 보고, 수평 연계

## 6. 커맨드센터 마이그레이션
- 이관 패키지, 세션 종료 시 저장

## 7. 병렬 처리 방식
- Wave 패턴, 병렬화 원칙

## 8. 작업 계획 방식
- 태스크 설계, 지시 원칙, 모니터링 원칙

## 9. 새 팀 빌드 체크리스트 (Phase 1~4)

## 10. 트러블슈팅
| 문제 | 원인 | 해결 |
|------|------|------|
| 에이전트 응답 없음 | 브릿지 죽음 | 재시작 |
| 메시지 무시됨 | 멘션 규칙 위반 | 올바른 토큰 사용 |
| 5분 타임아웃 | 작업이 너무 큼 | 분할 |
| 아카샤 401 | 세션 만료 | 재인증 |
