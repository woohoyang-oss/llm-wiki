# 서버 정상유무 체크리스트

## 배포 후 필수 검증 (모든 배포 시 수행)

### 1. 컨테이너 상태
```bash
docker ps --format '{{.Names}} {{.Status}}'
# 6개 전부 Up: esbot-admin, esbot-api, esbot-worker, esbot-beat, esbot-db, esbot-redis
```

### 2. HTTP 응답
```bash
curl -s -o /dev/null -w '%{http_code}' https://mail.tbe.kr/  # 200 확인
curl -s -o /dev/null -w '%{http_code}' http://localhost:8000/admin/dashboard  # 200 or 401
```

### 3. 프론트 렌더링
```bash
curl -s https://mail.tbe.kr/ | grep '<title>'
# '<title>ToBe auto email</title>'
```

### 4. Server Action 에러 확인
```bash
docker logs esbot-admin --tail 10 2>&1 | grep -i 'error'
# 'Failed to find Server Action' → --no-cache 리빌드 필요
```

### 5. API 에러
```bash
docker logs esbot-api --tail 10 2>&1 | grep -i 'traceback\|error\|500'
```

### 6. 메일 폴링
```bash
docker logs esbot-beat --tail 5 2>&1 | grep 'Sending due task'
```

## Server Action 에러 조치
```bash
cd /home/ec2-user/auto-email
docker-compose build --no-cache admin
docker-compose up -d admin
```

## 클라이언트 에러 (Application error: client-side exception)
브라우저 캐시와 새 빌드 충돌. Ctrl+Shift+R 또는 시크릿 모드로 접속.

## 검증 주체
- Tester: 배포 후 1~6번 전체 검증
- Reviewer: 코드 리뷰 시 Server Action 호환성 확인
- 커맨드센터: /loop 폴링에서 HTTP 모니터링
