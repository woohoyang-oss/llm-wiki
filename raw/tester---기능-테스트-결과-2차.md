# 브라우저 기능 테스트 2차 결과 (2026-03-23)

## auth_token 키 사용 + 새로고침 방식

### 결과: 5/5 여전히 로그인 화면

### 핵심 발견
- localStorage에 auth_token 주입 후 새로고침하면 **토큰이 사라짐** (hasToken: false)
- SPA 앱이 API 호출 → 401 응답 → auth_token 삭제 → 로그인 페이지 리다이렉트
- api-client.ts에서 auth_token 키 사용 확인됨 (7번째 줄)

### 원인 분석
1. JWT 토큰의 exp(만료) 시간이 이미 지났을 수 있음
   - iat: 1774250532 (2026-03-20 20:42:12 UTC)
   - exp: 1774336932 (2026-03-21 20:42:12 UTC)
   - **현재 시간 기준 이미 만료됨 (2일 전)**
2. SPA가 /api/auth/me 호출 → 서버에서 만료된 토큰 거부 → 401 → 토큰 삭제

### 결론
- 토큰 키(auth_token)는 맞음
- **JWT 토큰이 만료되어** 인증 실패
- 새로운 토큰 발급이 필요하거나, Google OAuth 로그인 플로우를 통해야 함

### 권장사항
- 새 JWT 토큰 발급 (exp를 현재 시간 이후로)
- 또는 프론트엔드 로그인 테스트는 Google OAuth 통한 실제 플로우로 진행
