# 코드 리뷰

$ARGUMENTS 를 리뷰하세요. 인자가 없으면 현재 git diff (staged + unstaged)를 리뷰합니다.

## 1단계: 변경 사항 파악

- 인자가 PR 번호인 경우: `gh pr diff $1`로 diff 확인
- 인자가 없는 경우: `git diff HEAD`로 변경 사항 확인
- 인자가 파일 경로인 경우: 해당 파일 읽기

## 2단계: 리뷰 관점

다음 관점에서 분석하세요:

### 보안
- SQL 인젝션, XSS, 커맨드 인젝션 등 OWASP Top 10
- 하드코딩된 시크릿, API 키
- 인증/인가 누락

### 버그
- 엣지 케이스 미처리
- Null/undefined 참조
- 리소스 누수 (미닫힌 커넥션 등)
- 동시성 이슈

### 성능
- N+1 쿼리
- 불필요한 반복/재계산
- 메모리 비효율

### 코드 품질
- 네이밍 명확성
- 함수 크기 및 책임 분리
- 중복 코드
- 에러 핸들링

## 3단계: 리포트 형식

```
🔍 코드 리뷰 결과

■ 요약
- 변경 파일: X개
- 발견 이슈: 심각 X / 주의 X / 개선 X

■ 심각 (반드시 수정) 🔴
1. [파일:라인] — [이슈 설명]
   제안: [수정 방법]

■ 주의 (수정 권장) 🟡
1. [파일:라인] — [이슈 설명]
   제안: [수정 방법]

■ 개선 (선택) 🟢
1. [파일:라인] — [이슈 설명]
   제안: [수정 방법]

■ 잘한 점 👍
- [긍정적인 부분 언급]
```
