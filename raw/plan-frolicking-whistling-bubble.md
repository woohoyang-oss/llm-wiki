# Auto-DM 서비스 기획서

## Context
ManyChat을 대체하는 자체 DM 자동응답 시스템. 브랜드별(키크론, 지티기어, 포스윙 등) 봇을 운영하며, 채널별(인스타, 텔레, 웹챗, 카카오) 확장 가능한 구조. 1단계는 **키크론 인스타그램 DM 자동응답**.

---

## 2. 핵심 도메인 모델

### Brand (브랜드/채널)
- 키크론, 지티기어, 포스윙 등
- 브랜드별 독립 설정 (톤앤매너, 지식베이스, 시나리오)

### Bot (봇)
- 브랜드 하위에 여러 봇 생성 가능
- 역할 지정: `simple_reply` | `cs` | `product_intro` | `event` | `guide`
- 봇별 LLM 모델 선택 (Claude / GPT / etc)
- 봇별 채널 연결 (인스타, 텔레, 카카오 등)

### Scenario (시나리오)
- 봇의 대화 흐름 정의
- 트리거 조건 → 응답 로직 → 데이터 수집 → 후속 액션

### Conversation (대화)
- 사용자별 대화 세션 추적
- 상태 관리 (진행 중, 완료, 에스컬레이션)

## 3. 트리거 시스템

### Instagram 트리거 (Graph API Webhook)
| 트리거 | 설명 | 예시 |
|--------|------|------|
| `keyword` | DM에 특정 키워드 포함 | "이벤트", "응모" |
| `story_mention` | 스토리에 멘션 | 브랜드 태그 시 자동 감사 DM |
| `story_reply` | 스토리에 리플라이 | 스토리 답장 시 대화 시작 |
| `comment` | 게시물 댓글 | 특정 게시물에 댓글 → DM |
| `any_dm` | 모든 DM 수신 | 기본 CS 봇 |
| `first_dm` | 첫 DM | 신규 고객 환영 메시지 |

### 트리거 우선순위
1. 키워드 매칭 (정확 > 부분 일치)
2. 이벤트/캠페인 연결 트리거
3. 기본 봇 (fallback)

## 4. Phase 1: 키크론 인스타 봇

### 4-1. 이벤트 대응 DM 봇
**목적**: 이벤트 응모를 대화형으로 처리 (폼 대체)

```
[트리거] 게시물 댓글에 "응모" or 스토리 멘션 or DM "이벤트"
    ↓
[봇] "안녕하세요! 🎉 키크론 OO 이벤트에 참여해주셔서 감사합니다!"
    ↓
[수집] 이름 → 전화번호 → 주소 → 사용 키보드 모델
    ↓
[확인] "입력하신 정보가 맞나요? [이름/전화/주소]"
    ↓
[완료] "응모 완료! 당첨자 발표는 OO일입니다"
    ↓
[저장] Google Sheets에 자동 기록
```

**수집 데이터 → Google Sheets 구조**:
| timestamp | ig_username | name | phone | address | event_name | status | custom_field_1 | custom_field_2 | ... |

**커스텀 필드**: 이벤트별로 수집 항목을 자유롭게 추가 가능 (예: "사용 키보드 모델", "참여 동기" 등). 대시보드에서 이벤트 생성 시 필드 정의.

### 4-2. 단순 반응 DM 봇
**목적**: 일반 문의에 자동 응답 (CS 목적)

- 제품 문의 → 제품 정보 + 구매 링크
- AS 문의 → AS 접수 안내
- 배송 문의 → 배송 조회 안내
- 기타 → AI가 맥락에 맞게 응답 (키크론 제품 지식 기반)

### Knowledge Base (키크론 봇 지식)
- 제품 카탈로그 (모델, 스펙, 가격)
- FAQ (자주 묻는 질문)
- AS 정책
- 현재 진행 중인 이벤트
- 구매 링크 (스마트스토어 등)

## 5. 기술 스택 (제안)

| 영역 | 기술 | 이유 |
|------|------|------|
| **서버** | Node.js (TypeScript) | Meta Graph API SDK 호환, 빠른 개발 |
| **프레임워크** | Next.js (App Router) | API Routes + 웹 대시보드 통합 |
| **DB** | PostgreSQL (Supabase) | 대화 이력, 봇 설정, 사용자 데이터 |
| **데이터 Export** | Google Sheets API | 이벤트 응모 데이터 자동 기록 |
| **LLM** | Claude / GPT (선택 가능) | 봇별 모델 설정 |
| **인스타 연동** | Instagram Graph API | Webhook으로 실시간 수신 |
| **배포** | Vercel or Railway | Webhook 엔드포인트 + 대시보드 |
| **인증** | NextAuth | 대시보드 로그인 |

## 6. 웹 대시보드 주요 화면

1. **봇 목록** — 브랜드별 봇 관리 (생성/수정/활성화)
2. **시나리오 편집기** — 트리거 + 대화 흐름 시각적 편집
3. **대화 모니터링** — 실시간 대화 확인, 수동 개입 가능
4. **이벤트 관리** — 진행 중인 이벤트, 응모 현황
5. **통계** — 대화 수, 응답률, 이벤트 참여율
6. **설정** — 채널 연결, API 키, LLM 설정

## 7. 확장 로드맵

| Phase | 내용 |
|-------|------|
| **1단계** | 키크론 인스타 DM (이벤트 응모 + 단순 CS) |
| **2단계** | 지티기어 인스타 추가, 웹 대시보드 고도화 |
| **3단계** | 포스윙 추가 (시나리오 가이드 + 드립 메시지) |
| **4단계** | 텔레그램/카카오 채널 확장 |
| **5단계** | 멀티채널 통합 관리, 고급 분석 |

## 8. 1단계 구현 범위 (MVP)

### 필수
- [ ] Instagram Graph API Webhook 연동 (메시지 수신/발신)
- [ ] 키워드 트리거 매칭 엔진
- [ ] 이벤트 응모 대화 플로우 (정보 수집 → Google Sheets 저장)
- [ ] LLM 기반 자유 응답 (키크론 지식베이스)
- [ ] 기본 웹 대시보드 (봇 설정, 대화 모니터링)

### 선택 (있으면 좋음)
- [ ] 스토리 멘션/리플라이 트리거
- [ ] 게시물 댓글 트리거 → DM 발송
- [ ] 대화 통계 대시보드

## 9. 검증 계획
1. Instagram Graph API 테스트 계정으로 Webhook 수신 확인
2. 키워드 트리거 → 이벤트 응모 플로우 end-to-end 테스트
3. Google Sheets 자동 기록 확인
4. LLM 자유 응답 품질 확인 (키크론 FAQ 기반)
5. 웹 대시보드에서 대화 모니터링 확인
