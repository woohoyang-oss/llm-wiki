# PI 번호별 이슈 업데이트 가이드

## 목적
발주·납기·단가 관련 이슈를 **PI(Proforma Invoice) 번호 단위**로 추적·업데이트하여, 건별 진행 상황을 정확히 파악하고 누락을 방지합니다.

---

## 기본 원칙

1. **모든 이슈는 PI 번호에 귀속시킨다** — PI가 아직 없는 건은 `PI미발행`으로 분류하되, PI 수령 즉시 번호를 부여한다.
2. **하나의 PI에 여러 이슈가 있을 수 있다** — 단가, 수량, 납기, 출하 등 이슈 유형을 구분하여 기록한다.
3. **상태값은 4단계로 통일한다** — ✅ 완료 / 🔄 진행중 / ⏳ 대기 / ❌ 이슈

---

## 업데이트 항목 (필수)

| 항목 | 설명 | 예시 |
|------|------|------|
| **PI 번호** | Proforma Invoice 번호 | PI-2026-0412 |
| **브랜드** | Keychron / MOZA / Playseat / Aiper / TrackRacer | Keychron |
| **모델명** | 해당 PI에 포함된 제품 | Q11 KR, K Ultra |
| **이슈 유형** | 발주수량 / 단가 / 납기 / 출하 / 기타 | 단가 |
| **상태** | ✅ 🔄 ⏳ ❌ | 🔄 진행중 |
| **상세 내용** | 현재 상황 요약 (1~2줄) | Max 기준 +5USD 협의 완료 |
| **담당자** | 내부 담당자 | 문소현 |
| **상대방 담당자** | 본사/거래처 담당자 | Nick, Liana |
| **최종 업데이트일** | 마지막 갱신 날짜 | 2026-04-06 |
| **예상 완료일** | 해결 예상 시점 | 2026-04-09 |
| **비고** | 참고 사항, 연결 이슈 등 | 이사님 소통 중 |

---

## 업데이트 방법

### 1. 일일 보고 시
- 기존 브랜드별 정리 → **PI 번호를 반드시 명기**
- PI 미수령 건은 모델명으로 임시 키를 사용하되, PI 번호란에 `미수령` 표기

### 2. 작성 포맷 (텍스트 보고 시)

```
[브랜드] PI번호 | 모델명
- 이슈유형: 상태 | 상세내용
- 담당: 내부 / 외부
- 예상완료: YYYY-MM-DD
```

**예시:**
```
[Keychron] PI-2026-0389 | K Ultra
- 단가: ✅ 완료 | Max +5USD 확정
- 발주수량: 🔄 진행중 | 수량 확인 중
- 담당: 문소현 / Nick
- 예상완료: 2026-04-08
```

```
[Keychron] PI미수령 | EX75-A6, B1P, B6P, Q6 Ultra, J8, F5, F4
- 출하: ⏳ 대기 | 생산 안배 완료, PI 미수령
- 담당: 문소현 / Liana
- 예상완료: 확인 중
```

### 3. 상태 변경 시 필수 기재 사항
- **이전 상태 → 변경 상태** (예: 🔄→✅)
- **변경 사유** (예: Nick 메일 회신으로 단가 확정)
- **변경일자**

| Keychron | M6S | 납기 확인 | 🔄 | 확인 중 | Liana 대응 |

## 주간 리뷰 체크리스트

- [ ] 모든 🔄 진행중 건의 예상완료일 경과 여부 확인
- [ ] PI 미수령 건 중 1주 이상 경과 건 에스컬레이션
- [ ] ✅ 완료 건 아카이빙 (완료일 기준 2주 후 별도 이동)
- [ ] 신규 PI 수령 건 등록 및 기존 임시 키 매칭
- [ ] 브랜드별 담당자 커뮤니케이션 로그 확인
