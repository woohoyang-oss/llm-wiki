# 챗봇(우측 하단) 기능 완성도 점검 계획

## 목표
auto-email 플랫폼의 챗봇이 모든 관리 기능을 자연어로 수행할 수 있는지 점검.
엔드포인트/API/스킬이 챗봇에 연결되어 있는지 확인.

## 점검 항목
1. 특정 메일에서 규칙 만들기
2. 특정 메일에서 자동화 하기
3. 담당자 변경/배정
4. 고지문(서명) 변경
5. 지식 문서 추가/수정
6. 매크로 관리
7. 배정 규칙 관리
8. 룰 관리
9. 템플릿 관리
10. SLA 관리
11. 태그 관리
12. 메일 검색/조회
13. 통계/대시보드 조회
14. 승인 큐 관리

## 병렬 분석 계획

### Wave 1: 현황 분석 (병렬)
- Coder: 챗봇 관련 소스코드 분석 (chatbot 컴포넌트, API 연결, 스킬 정의)
- TL: 챗봇이 지원해야 할 기능 목록 정의 (사용자 시나리오별)
- Reviewer: 현재 admin API 엔드포인트 전수 조사 → 챗봇에서 호출 가능한지
- Tester(Slack): 실제 챗봇에 자연어 명령 테스트
- Tester(아카샤): 챗봇 관련 프론트엔드 코드 분석 (skills, tools 정의)
- PL: 전체 취합

### Wave 2: GAP 보충 (분석 결과 기반)
- 누락된 스킬/엔드포인트 추가 구현
- 자연어 인텐트 매핑 보강
- 테스트 + 배포

## 핵심 질문
- 챗봇이 admin.py의 모든 CRUD 엔드포인트를 호출할 수 있는가?
- 자연어 → API 매핑이 정의되어 있는가?
- 컨텍스트(현재 보고 있는 메일 등)를 챗봇에 전달할 수 있는가?
